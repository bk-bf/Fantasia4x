# LINEAGES тАФ ancestral-blood mutation trees тЬЕ COMPLETE 2026-07-09

> **Related:** [LINEAGES-II](LINEAGES-II-2026-07-10.md) (successor: transforms, feeding, heritage flatten) ┬╖ [TRAITS](TRAITS-2026-07-10.md) ┬╖ [game/DESIGN](../../game/DESIGN.md) ┬╖ [game/DECISIONS](../../game/DECISIONS.md) ┬╖ [ROADMAP](../open/ROADMAP.md) ┬╖ [RACE-SYSTEM (archived)](RACE-SYSTEM-2026-07-13.md)

## ┬з1 ┬╖ Concept

A **lineage** is an ancestral bloodline (Beast, Werewolf, Vampiric, Amphibian, Arachnid, plus the old
heritages тАФ Stoneblood, Farseer, Crustacean, BlightedтАж). It is a *collection of linked traits a pawn can
grow into over its life*, ├а la a CDDA mutation tree тАФ **not** a bundle a pawn gets all at once. Lineages
run across [Cultures](RACE-SYSTEM-2026-07-13.md) (a Culture is the present-day formation; a lineage is the
blood underneath, individual to a pawn), so they are per-pawn, not per-Culture.

Two ways a pawn ends up in a lineage:

1. **Born into it** тАФ a Culture guarantees the lineage's parent marker + one random member trait at
   generation (the old `guaranteedTraits` roll, [Culture.ts:435](../../../src/lib/game/core/Culture.ts)).
2. **Awakened into it** тАФ a pawn rolls a standalone gateway trait (`lineageExclusive: false`) with no
   parent, then earns the parent over its life by doing lineage-fitting **deeds** (┬з4).

Once a pawn has the parent marker, it can **grow new lineage traits** and **evolve staged ones** at its
seasonal growth events. This replaces the closed nested-`subCapabilities` bundle.

## ┬з2 ┬╖ Schema тАФ three independent trait properties

Replaces the nested `subCapabilities` on heritage banners. Each is orthogonal:

- [x] **`lineage?: string[]`** тАФ the lineage pools this trait belongs to (an **array** тАФ Feral Adrenaline
      is `["beast","werewolf"]`, both draw it). A member pawn can gain any trait whose `lineage` includes
      its lineage id. *(field added; content tagging in Phase 2)*
- [x] **`lineageExclusive?: boolean`** (default `true`) тАФ if `false`, the trait can roll **standalone**
      from the Culture pool with no parent (the gateway traits: claws, spider-eyes, gills, venom, fur).
- [x] **`awakens?: string[]`** тАФ candidate **awakening conditions** (┬з4) a standalone gateway carries. A
      trait can be a lineage member without being standalone, and standalone without awakening (Feral
      Adrenaline is a shared pool trait, not a gateway).
- [x] **`conflictGroup?: string`** on branches тАФ mutually-exclusive lineage forks (robust-skin branch vs
      ranged branch) so a lineage genuinely diverges per pawn. Checked in `gainableMembers`.
- [x] Parent marker traits are PURE identity markers тАФ all five new lineages ship clean markers
      (`beast-heritage`, `werewolf-heritage`, `amphibian-heritage`, `vampiric-heritage`, `arachnid-blood`)
      with their stat baselines as `stat`-kind members (Bestial Might, Cold Vigor, Unnatural Vigor,
      Spider's Poise). The OLD heritage bundles (StonebloodтАж) flatten in [LINEAGES-II ┬з4](LINEAGES-II-2026-07-10.md).

## ┬з3 ┬╖ Growth-event hook

Every pawn already banks ~4 growth offers/year (3 seasonal + 1 doubled birthday,
[PawnGrowthService.processDay](../../../src/lib/game/services/PawnGrowthService.ts)). Lineage progression
rides the SAME event. On each growth event, in priority order:

- [x] **Awakening meter full?** тЖТ grant the lineage parent (the pawn "turns"), grant its **first** lineage
      trait immediately (the payoff тАФ awakening must feel rewarding, not anticlimactic), clear the meter.
- [x] Else **evolve a staged trait** the pawn has (S1тЖТS2тЖТS3 via `evolvesTo`) тАФ ~10%, checked first /
      slightly favoured over new-trait gain.
- [x] Else, **if the pawn has a lineage parent**, gain a *new* member trait from that lineage тАФ ~10%.

*Implemented: `Lineages.lineageGrowthEvent`, called from `PawnGrowthService.bankOffer` at each growth
event; a newly-granted trait's one-shot effects (stat/graft/bodyMod) applied by `Pawns.applyGainedTrait`.*

A standalone gateway with no parent can only evolve its own stage (no new-trait gain) until it awakens.

## ┬з4 ┬╖ Awakening meter

A standalone gateway tracks progress toward **тЙе2 lineages at once**; the player steers by which deeds the
pawn performs, and whichever meter fills first is the one that awakens (choice of path, not a pre-assigned
single meter).

- [x] **Deeds** тАФ `pawn.deeds: Record<string, number>` counter map + live increment hooks:
  - EAT (`recordMealDeeds`): `ateRawMeat` / `ateCarcass` / `ateCanineMeat`.
  - KILL (`creditKillDeeds` in Combat): `unarmedBigKill` (unarmed/natural weapon vs bodyScale тЙе 1.3),
    `kill:canine`, `kill:arachnid`, `venomKills` (victim died envenomed), `ensnaredKills` (died held).
  - FEED (`applyOneOnHitEffect`): `drewHumanoidBlood` on a successful blood-drain vs a humanoid.
  - ENV (hourly, gated on `lineagePaths`): `wetHours` (wetness тЙе 50).
  - Remaining hooks (moonlight, waterHours, fishing, sleep-unsheltered, raw-diet-days, ateFish/ateInsect,
    nightKills) need machinery that lands with their lineages тЖТ [LINEAGES-II](LINEAGES-II-2026-07-10.md).
- [x] **Targets** roll per pawn within a **high** range (тЙИ2 seasons) тАФ `seedAwakeningPaths` rolls in the
      awakening def's `range`.
- [x] **Regression** тАФ the meter decays past a grace window (`advanceAwakeningMeters`, per day); paused
      while actively progressing.
- [x] **Reveal** тАФ `AwakeningMeters.svelte` in the pawn NEEDS panel shows each meter (lineage ┬╖ %/READY ┬╖
      the deed label) only while `value > 0`; hidden when it lapses to empty. A FULL meter LOCKS (never
      decays) until the growth event consumes it.
- [x] **No creature-awakening** тАФ awakening comes ONLY from a born parent or a standalone gateway (no
      lineage-S1 on creatures; `seedAwakeningPaths` only seeds from a pawn's own gateway traits).
