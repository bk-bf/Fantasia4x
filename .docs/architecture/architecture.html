In the code

The single value is uiState.currentScreen. A plain list of “if it's this, show that” in +page.svelte picks the screen; a tab click calls setScreen().

In the gameMarking the oak becomes the request designate(chop, tile 14,9). It crosses the wall, gets looked up in the list, and now that tile is flagged for chopping. That request is the only way the world learned about your intent.
WHY IT'S BUILT THIS WAY

One doorway, three users: the buttons you click, the automated test scripts, and the dev tools all send the exact same requests. Same requests in the same order always produce the exact same game.

In the code: the list is COMMANDS in commands.ts (88 entries) · chop = designate, equip = equipPawnItem, draft = toggleDraft.

4Two folders that look similar

Does it drive the clock, or does it answer a question?

Drives the clock — systems/

The few things that push time forward or reach across everything at once:

  • the turn-taker gameEngine
  • each pawn's moment-to-moment state PawnStateMachine
  • the blow-by-blow of fights Combat
  • totalling up every bonus ModifierSystem

Answers questions — services/

Everything else. Give it the world and a question, it hands back an answer or an updated world. Each stays inside its own topic; the drivers on the left call on them.

In the game“Is there a chop job near this pawn?” is a question → a service. “Advance this pawn's axe-swing by one tick” drives the clock → a system.

The rulebook is data, not code

Creatures, items, recipes, traits, buildings — all live in big data files database/*.jsonc. Adding content usually means editing data, not writing logic.

The shapes of everything

What a pawn, a building, an item is — the definitions live in core/types/.

Heads-up: the old notes point at core/Items.ts and Buildings.ts — those are gone. It's all database/*.jsonc now.

Four rules that hold everywhere

When there's one right way, there's only one way

“How good is this pawn at this job?”

One place, from body and traits — no second opinion. A strong, axe-trained pawn fells the oak in half the swings. getWorkModifiers

“What bonuses apply right now?”

One tallier adds up gear, buildings and traits, and shows its receipts. Your sharpened axe's +2 shows up here. ModifierSystem

“How do things move across the map?”

Heavy path-finding math, in Rust for speed, behind one tidy door. The pawn routes around the pond to reach the oak. PathfinderService → WASM

“Can it run with no window open?”

Yes — the same game, driven by scripts. A test fells 1000 oaks with no window, just to check the wood adds up. headless sim

Each has a “don't build a second one of these” rule burned in — which is why a new feature is usually new data, not a new code path.
